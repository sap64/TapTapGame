Daniel Nasello's Mobile Anination/Transition Library
  • Read First

    MobileTrans.js is a mobile transitions library that does everything from transitioning between two screens or showing modal screens/pop ups. To use this library, there are some things that need to be understood first. ONLY ELEMENTS WITH POSITION ABSOLUTE can have animations applied to them. If you try to apply an animation to an element that does not have position absolute it will not work correctly. This Library Replaces any classes previously applied to the element. Meaning... if you have styles such as color, background-color, height width etc, they should all be applied to the id of the element.

    Boilerplate styles. any element that will act as a screen of your application should have the following styles applied to them:

           
               #element{
    
                     position:absolute;
    
                      height:100%;
    
                       width:100%;
    
                      display:none;
    
                }

    The library will handle the showing and hiding of elements. All screens except for your first screen should be hidden at start, and the library will do the rest

    Again, this is only for elements that will be used as screens. If you want to transition smaller elements, you may, however, make sure all styles are applied to the id and not the class

    NOTE: all elements must be called by their ID. IE, if you want to fade an element out, you must either know or rertrieve its id in order for the library to fade it out. All exampled below are demonstrated with an elements ID

    How it works:

    Using JavaScript to change or "trigger" changes in the Class of HTML elements that have CSS animations attached to them. Lets look at the following:

               
          
    @-webkit-keyframes cf3FadeIn {
    
	  0% {opacity:0;}
    
	  100% { opacity:1;}
    
	}
    	
    

                
        .fadein{
    
	   -webkit-transform: translate3d(0px,0px,0px);
    
	   -webkit-perspective:1000;
    
	   -webkit-backface-visibility:none;
    
	   -webkit-animation-name: cf3FadeIn;
    
            -webkit-animation-iteration-count: 1;
    
            -webkit-transform hardware acceleration;
    
            -webkit-animation-duration: .4;
    
            -webkit-animation-direction: normal;
    
            opacity:1;
    
             z-index:998;
    
    }

    Lets take a look at the class "fadein". this is the class applied to an element if we want our element to start at opacity 0, and keyframe to opacity 1 (from completely transparent to visible.

    Notice Three things about the fade in class.Not only do i have the "keyframe above linked", I also defined "opacity:1".

    Why this is imperitive: Becuase when keyframe animations are done being performed, elements go back to their original or "resting state". ie once the fadein animation is complete, it would go back to being invisible. To prevent this, I define opacity:1 in my new fade in class in order to keep the state and make sure the transition/animation is one way and will stay "as is" when done. This same rule applies to all my classes whether it be translation rotation skew opacity etc.

    Notice the use of parameters. None of the element ids are hardcoded. This allows us to dynamically change the "context" of the function in order for us to transition in/out any screen/element as we see fit

  •            
            fade("out","in");
    This function takes two elements and fades one in and one out. For example, I am on a page with an id of screen_one and I want to transition to an element with an id of screen_two I would simply call 
               
              fade("screen_one","screen_two");
  •            
            slideleft("out, "in");
    This function takes two elements and slides the "incoming element in from the right and slides the outgoing div to the left using translation. For example, I am on a page with an id of screen_one and I want to transition to an element with an id of screen_two I would simply call 
               
               slideleft("screen_one","screen_two");
   
  •            
            slideright("out" , "in");
    This function takes two elements and slides the "incoming element in from the left and slides the outgoing div to the right using translation. For example, I am on a page with an id of screen_one and I want to transition to an element with an id of screen_two I would simply call 
               
             slideright("screen_one","screen_two");
  •            
              slideup("in","out");
    This function takes two elements and slides the "incoming element in from the bottom and slides the outgoing div going up using translation. For example, I am on a page with an id of screen_one and I want to transition to an element with an id of screen_two I would simply call 
               
              slideup("screen_one","screen_two");
  •            
                slidedown("in","out");
    This function takes two elements and slides the "incoming element in from the top and slides the outgoing div going downward using translation. For example, I am on a page with an id of screen_one and I want to transition to an element with an id of screen_two I would simply call 
               
               slidedown("screen_one","screen_two");
  •            
            shake("shake");
    This function takes one element and applies the shake class to it which has a key-frame animation to emulate the phone rumbling or shaking For example if you have a password screen and the user enters an incorrect passowrd you can make the password_screen shake like so 
               
             shake("password_screen")
  •            
             fadeIn("in")
    This function takes one screen and fades it in. For example, you have an image or some sort of popout/modal screen that you would like to bring into the document with a smooth fade transition you would do it like so: 
               
              fade("one");
  •            
             fadeOut("out");
    This animation takes an element and fades it out of sight and hides it For example, we want to hide or dismiss a popout or modal screen that we faded in. we can then fade it out like so: 
               
              fadeOut("out");
  •            
              slideoneup("in");
    This animation allows us to slide a modal screen from the bottom of the page up to the top For example, I click a thumbnail of a picture and I want to slide up another div to see it full screen. I would do it like so: 
               
              slideoneup("fullview");
  •            
            slideonedown("out");
    This animation allows us to slide down an element and ultimately hide it from view For example, I am done looing a picture full screen and I want to dismiss the modal view. I would do it like so: 
               
             slideonedown("fullview")
